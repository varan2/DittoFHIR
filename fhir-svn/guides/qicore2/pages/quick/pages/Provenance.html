Provenance

  • Who, What, When for a set of resources

    Base StructureDefinition for Provenance Resource

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        activity 0..1 Code
      An activity is something that occurs over a period of time and acts upon or with entities; it may include consuming, processing, transforming, modifying, relocating, using, or generating entities.

      (Extensible)
        agent 1..* List<Agent>
      An actor taking a role in an activity for which it can be assigned some degree of responsibility for the activity taking place.
        entity 0..* List<Entity>
      An entity used in this activity.
        location 0..1 Location
      Where the activity occurred, if relevant.
        period 0..1 interval<DateTime>
      The period during which the activity occurred.
        policy 0..* List<Uri>
      Policy or plan the activity was defined by. Typically, a single activity may have multiple applicable policy documents, such as patient consent, guarantor funding, etc.
        reason 0..* List<Code>
      The reason that the activity was taking place.

      (Extensible)
        recorded 1..1 DateTime
      The instant of time at which the activity was recorded.
        signature 0..* List<Signature>
      A digital signature on the target Reference(s). The signer should match a Provenance.agent. The purpose of the signature is indicated.
        target 1..* List<Resource>
      The Reference(s) that were generated or updated by the activity described in this resource. A provenance can point to more than one target if multiple resources were created/updated by the same activity.
